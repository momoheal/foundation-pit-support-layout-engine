# 基坑内支撑模块概要设计

## 1. 总体目标

本概要设计基于 `docs/strut_module_requirements.md`，用于指导内支撑模块从几何原型升级为可选择支撑体系、可校核几何合法性、可统计支撑用量、可输出工程可读 DXF 图纸的布置工具。

已确认设计口径：

- 本文档输出位置为 `docs/high-level-design.md`。
- 支撑统计中，“对撑”作为“主撑”的主要形式处理；统计模型保留主撑总量，并可在明细中标注对撑形式。

第一阶段设计重点不包含完整结构内力计算、截面承载力验算、多层支撑竖向体系、施工图图框和商业 CAD 双向编辑同步。数据结构应为后续力学分析预留节点、构件、类型、长度、连接关系和支撑体系信息。

## 2. 模块划分

内支撑模块划分为以下核心模块：

| 模块 | 主要职责 | 对应需求 |
| --- | --- | --- |
| GUI 参数输入模块 | 读取 DXF 边界，收集支撑体系、间距、偏移、安全净距、材料、圆环撑、桁架等参数 | F1, F2 |
| 参数治理模块 | 统一默认值、兼容旧参数、校验缺失参数和未知支撑体系 | F1, F2, N3, N4 |
| 支撑体系分发模块 | 根据 `support_system` 调用正交对撑、八字撑/角撑、直边桁架、内圆型支撑策略 | F1, F5, F9, F10 |
| 几何生成策略模块 | 生成腰梁、主撑、角撑/八字撑、连杆/腹杆、桁架、圆环撑、径向撑和立柱候选节点 | F5-F10 |
| 节点与构件模型模块 | 维护 `nodes`、`members`、`outlines`、`stats`，并兼容旧 layout 字段 | F3, F8, F12, N3, N5 |
| 几何校核模块 | 校核非法非端点交叉、连接点间距、核心筒保护区侵入和构件统计 | F3, F4 |
| 统计模块 | 统计主撑、角撑、腹杆、连杆、圆环撑、径向撑、总长度和立柱数量 | F3, F12 |
| DXF 输出模块 | 按图层输出边界、腰梁、中心线、调试图层和后续支撑边线 | F11, F13 |
| 命令行测试模块 | 提供不依赖 GUI 的最小回归测试入口 | F4, N2 |

## 3. 模块关系

模块之间采用“输入参数 -> 生成策略 -> 拓扑模型 -> 校核与统计 -> DXF 输出”的流水线关系。

```text
DXF 边界 + GUI 参数
        |
        v
参数治理模块
        |
        v
支撑体系分发模块
        |
        +--> 正交对撑策略
        +--> 八字撑/角撑策略
        +--> 直边桁架策略
        +--> 内圆型支撑策略
        |
        v
节点与构件模型
        |
        +--> 几何校核模块
        +--> 统计模块
        |
        v
兼容 layout + DXF 绘图实体
        |
        v
GUI 预览 / DXF 输出 / 命令行测试
```

核心约束：

- GUI 不直接实现支撑生成算法，只负责输入、调用和输出。
- `StrutEngine` 不依赖 Tkinter，应可被命令行测试直接调用。
- 几何生成不直接面向 DXF 图层，而应先输出节点、构件、轮廓和统计数据。
- 旧字段 `waling`、`corners`、`struts`、`ties`、`pillars` 短期保留，用于兼容现有绘图逻辑。

## 4. 核心数据模型

### 4.1 参数模型

统一参数模型至少包含：

| 参数 | 含义 | 默认值或约束 |
| --- | --- | --- |
| `support_system` | 支撑体系 | `orthogonal` / `brace` / `straight_truss` / `circular` |
| `spacing_min` | 推荐最小连接间距 | 6.0m |
| `spacing_max` | 推荐最大连接间距 | 9.0m |
| `spacing` | 兼容旧目标间距 | 9.0m |
| `waling_offset` | 腰梁相对基坑边界内偏移 | 按米 |
| `safe_dist` | 构件或立柱到边界/腰梁的安全净距 | 按米，必须参与计算或校核 |
| `strut_material` | 支撑材料 | 钢支撑或混凝土支撑 |
| `main_width` | 主撑宽度 | 按米 |
| `tie_width` | 连杆宽度 | 按米 |
| `waling_width` | 腰梁宽度 | 按米 |
| `pillar_min_spacing` | 立柱最小间距 | 默认采用 `spacing_min` |
| `enable_haunch` | 是否启用八字撑 | 布尔值 |

圆环撑额外参数：

- `core_center`
- `core_diameter`
- `core_clearance`
- `ring_edge_clearance`
- `radial_spacing_min`
- `radial_spacing_max`
- `radial_count`

桁架额外参数：

- `truss_depth`
- `truss_panel_min`
- `truss_panel_max`
- `truss_web_with_main`
- `truss_web_without_main`

### 4.2 节点模型

节点表示结构连接、构造连接和后续立柱筛选候选点。

```python
{
    "id": "N001",
    "pos": (x, y),
    "kind": "strut_cross",
    "source": ["M001", "M002"],
    "system": "orthogonal"
}
```

典型节点类型包括：

- `waling_point`：腰梁连接点。
- `strut_end`：主撑端点。
- `strut_cross`：主撑交点。
- `corner_end`：角撑/八字撑端点。
- `tie_end`：连杆端点。
- `truss_node`：桁架节点。
- `ring_radial`：圆环撑与径向撑连接点。

### 4.3 构件模型

构件表示参与计算、校核、统计和绘图的支撑对象。

```python
{
    "id": "M001",
    "kind": "main_strut",
    "system": "orthogonal",
    "start": "N001",
    "end": "N002",
    "geometry": [(x1, y1), (x2, y2)],
    "width": 0.8,
    "material": "steel"
}
```

构件类型至少包括：

- `waling`
- `main_strut`
- `corner`
- `haunch`
- `tie`
- `truss_chord`
- `truss_web`
- `ring_strut`
- `radial_strut`

### 4.4 输出模型

`StrutEngine.solve()` 输出应逐步升级为：

```python
{
    "waling": [],
    "corners": [],
    "struts": [],
    "ties": [],
    "pillars": [],
    "nodes": [],
    "members": [],
    "outlines": [],
    "stats": {},
    "issues": []
}
```

旧字段用于兼容 DXF 输出；新字段用于后续策略、校核、统计和测试。

## 5. 支撑体系策略

### 5.1 支撑体系分发

`support_system` 是用户显式选择项。自动推荐只能作为提示，不替代用户确认。

建议分发逻辑：

```python
support_system = params["support_system"]

if support_system == "orthogonal":
    solve_orthogonal()
elif support_system == "brace":
    solve_brace()
elif support_system == "straight_truss":
    solve_straight_truss()
elif support_system == "circular":
    solve_circular()
else:
    raise ValueError("unsupported support_system")
```

### 5.2 正交对撑

正交对撑策略生成腰梁、主撑、主撑节点和基础立柱候选点。

设计要求：

- 腰梁为闭合折线。
- 主撑端点位于腰梁内部边界或合法节点上。
- 主撑连接点沿腰梁尽量均匀，推荐间距为 6m 到 9m。
- 主撑之间的交点可作为合法结构节点和立柱候选点。
- 阳角薄弱位置通过两个方向对撑或合理节点加强。

### 5.3 八字撑/角撑

八字撑/角撑策略在已有主撑、腰梁和节点基础上生成斜向构件。

设计要求：

- 候选斜撑生成后必须先经过相交校核。
- 与已有主撑、连杆、桁架和角撑发生非法非端点交叉时，应跳过、截断或改接到合法节点。
- 阴角优先采用角撑处理。
- 大基坑角部可使用角部桁架带替代单线角撑，避免单线角撑和桁架叠加。
- 该体系不自动生成完整直边桁架网络，但大基坑角部可生成局部角撑桁架。

### 5.4 直边桁架与边桁架网络

直边桁架策略应从“局部两撑之间的桁架”升级为“桁架化支撑网络”。真实工程中，围护支撑往往以连续边桁架带、内部多跨主桁架、三角腹杆网络和局部连杆共同工作。该策略应优先生成桁架体系，而不是在正交对撑结果上补少量斜杆。

设计要求：

- 腰梁本身作为边桁架外弦，内侧按 `truss_depth` 偏移生成内弦，腹杆形成三角形单元。
- 在坑内长向或主要受力方向生成多条主桁架带，主桁架之间通过连杆或腹杆连接。
- 有主撑区域优先使用 Warren 型腹杆。
- 无主撑区域使用 K 型腹杆。
- 连杆仅连接相邻主桁架、主撑或合法节点，不跨越无节点语义区域。
- 腹杆间距尽量落在 6m 到 9m。
- 桁架节点进入统一节点表。
- 桁架构件参与碰撞校核和长度统计。
- 纯对撑和八字撑/角撑体系不自动生成边桁架。

建议生成流程：

1. 根据腰梁闭合线生成边桁架带：腰梁线作为外弦，内弦按 `truss_depth` 偏移，沿边长按 `truss_panel_min/max` 分段。
2. 对每个边桁架分段生成 Warren 或 K 型腹杆，形成连续三角单元。
3. 根据基坑包围盒长短向和可用区域生成多条内部主桁架轴线。
4. 在相邻主桁架之间生成短向连杆或斜向腹杆，端点吸附到主桁架节点。
5. 对大开口、核心筒或环撑区域进行避让；必要时将主桁架改接到环外节点。

`tie` 与 `truss_web` 的划分：

- 桁架内部斜杆、K 型杆、Warren 腹杆记为 `truss_web`。
- 相邻主桁架之间的共同受力联系杆记为 `tie`。
- 若一个联系杆成为三角桁架的一部分，优先记为 `truss_web`；若它只是连接两条主桁架的次级联系，记为 `tie`。

### 5.5 内圆型支撑

内圆型支撑策略由用户显式选择，并依赖核心筒参数。

设计要求：

- `core_center` 通过 DXF 选点获得。
- `core_diameter` 由用户输入。
- 圆环为一圈真圆。
- 圆环半径不得小于 `core_diameter / 2 + core_clearance`。
- 圆环与核心筒外接圆之间至少保留 2m 净距。
- 圆环内部核心筒保护区必须完全空置。
- 径向撑只连接圆环外侧和外部支撑系统，不穿过圆环内部。
- 不规则基坑不自动采用内圆型支撑。
- 八边形、近圆多边形和用户显式选择 `circular` 的异形基坑可生成圆环支撑；测试应覆盖八边形圆环用例。
- 圆环外侧宜通过径向撑、斜向三角桁架或边桁架节点接入腰梁，不应只生成穿越中心的米字线。

### 5.6 工程图样本几何特征

根据参考图样，本轮迭代采用以下设计口径：

- 支撑平面图的主要视觉和结构特征应是桁架网络，而不是稀疏单线。
- 边界周边应有连续或分段连续的边桁架，尤其在长边、斜边、八边形切角和异形边界处。
- 内部支撑由多条主桁架带组成，主桁架之间通过可解释的连杆形成共同受力。
- 开口和核心筒周边采用环形支撑或多边形环梁，环外通过三角桁架接入边桁架。
- 立柱由拓扑候选筛选生成，不等同于所有桁架节点；普通桁架 panel 节点默认不直接生成立柱。
- 生成策略应尽量产生三角单元；矩形方格只能作为节点定位辅助，不应成为主要支撑形态。

## 6. 几何校核流程

几何校核模块应独立于 GUI 和 DXF 输出，接收 `nodes`、`members`、`rings`、核心筒保护区和参数范围。

校核流程：

1. 收集待校核构件，包括主撑、角撑、八字撑、连杆、腹杆、圆环撑、径向撑。
2. 建立端点和合法节点索引。
3. 检查构件之间的交点。
4. 判断交点是否为合法端点连接或允许的结构节点。
5. 报告非法非端点交叉。
6. 检查腰梁连接点间距是否接近 6m 到 9m。
7. 检查核心筒保护区是否被任何构件侵入。
8. 输出校核问题和统计结果。

校核问题应包含：

- 构件类型。
- 构件索引或 ID。
- 交点或违规坐标。
- 违规原因。
- 涉及的支撑体系。

允许的合法交叉示例：

- 主撑与主撑交点可作为结构节点。
- 圆环撑与径向撑交点可作为结构节点。
- 腹杆端点与桁架节点连接合法。

禁止的交叉示例：

- 角撑与主撑发生未节点化的非端点交叉。
- 连杆之间发生未节点化的非端点交叉。
- 任何支撑穿过核心筒保护区。

## 7. DXF 输出

DXF 输出模块从兼容 layout 和绘图实体中生成图层，不应反向驱动算法模型。

基础图层：

| 图层 | 内容 |
| --- | --- |
| `WALL_REF` | 基坑边界参考 |
| `WALING` | 腰梁中心线 |
| `STRUT` | 主撑中心线，包含对撑形式 |
| `CORNER` | 角撑/八字撑 |
| `TIE` | 连杆 |
| `TRUSS_WEB` | 桁架腹杆 |
| `RING_STRUT` | 圆环撑 |
| `RADIAL_STRUT` | 径向撑 |
| `PILLAR` | 立柱 |

增强图层：

- `STRUT_OUTLINE`
- `WALING_OUTLINE`
- `TRUSS_CHORD`
- `DEBUG_NODE`
- `DEBUG_ISSUE`
- `CORE_PROTECTION`

当前阶段可优先输出支撑边线和中心线。完整闭合外包轮廓属于后续增强，但数据模型应通过 `members.geometry + width` 为其预留生成入口。

## 8. 统计输出

统计模块从 `members` 和 `pillars` 生成 `stats`。

统计项至少包括：

| 字段 | 含义 |
| --- | --- |
| `main_strut_length` | 主撑长度，包含对撑作为主要形式 |
| `corner_length` | 角撑/八字撑长度 |
| `truss_web_length` | 桁架腹杆长度 |
| `tie_length` | 连杆长度 |
| `ring_strut_length` | 圆环撑长度 |
| `radial_strut_length` | 径向撑长度 |
| `total_support_length` | 支撑总长度 |
| `pillar_count` | 立柱数量 |

如需保留对撑明细，可在 `main_strut_breakdown` 中记录对撑形式、方向或支撑体系，但对撑不作为与主撑并列的总类。

## 9. 测试设计

命令行测试入口为 `python test_strut_minimal.py`。

最小测试用例：

| 用例 | 重点校核 |
| --- | --- |
| `rect_60x40` | 腰梁、主撑、角撑/八字撑、立柱、连接点间距、非法交叉、统计 |
| `l_shape` | 腰梁内偏移、凹角角撑、主撑裁剪、凹角非法穿越 |
| `octagon_cut` | 切角角撑、主撑与角撑合法节点、转角点避让 |
| `irregular` | 不崩溃、不自动采用内圆型支撑、至少生成腰梁和主撑、报告非法交叉 |
| `circle_or_ellipse_with_core` | 真圆圆环、核心筒净距、核心筒保护区、径向撑连接 |
| `octagon_circular_core` | 八边形或切角近圆基坑显式选择圆环支撑，检查圆环、径向撑和保护区 |
| `straight_truss_network` | 连续边桁架、多主桁架、多三角单元和合法连杆 |

每个用例输出：

- 构件数量。
- 分类长度统计。
- 校核结果。
- 失败时的构件类型、索引、坐标和违规原因。

测试失败应返回非零退出码。

## 10. 分阶段实现计划

### P0：测试和参数基础

1. 修复 `strut_validation.py` 中 `is_connection_point()` 缺失问题。
2. 跑通 `test_strut_minimal.py`。
3. 统一 `main_strut.py` 真实运行路径和测试路径的参数命名。
4. 增加 `support_system` 参数和 GUI 入口。
5. 增加 `spacing_min`、`spacing_max` 和连接间距校核入口。
6. 增加 `stats` 数据结构。

### P1：正交支撑和工程规则可用

1. 稳定正交对撑生成。
2. 修复角撑/八字撑与主撑非法交叉。
3. 修复连杆/腹杆方向策略。
4. 增加腰梁连接点推荐间距校核。
5. 初步从节点生成立柱。
6. 输出分类支撑长度统计。
7. 保持 DXF 图层稳定。

### P2：扩展支撑体系

1. 实现直边桁架支撑。
2. 实现内圆型支撑显式选择。
3. 实现核心筒保护区硬约束。
4. 输出支撑边线。
5. 完善节点、构件和轮廓模型。

### P3：力学分析预留和增强

1. 建立节点-构件拓扑图。
2. 增加以轴力为主的简化判断。
3. 增加圈梁连接点弯矩风险提示。
4. 增加支撑用量优化建议。

## 11. 未决问题

需求文档中“主撑”和“对撑”的统计口径已确认：对撑是主撑的主要形式，不作为与主撑并列的总类。

后续详细设计阶段仍需确认：

1. `support_system=brace` 时，是否默认启用八字撑，或继续由 `enable_haunch` 独立控制。
2. 角撑候选与已有构件非法交叉时，优先采用跳过、截断还是改接节点。
3. 连杆策略是否只允许单向连杆，或允许双向连杆并把交点节点化。
4. 立柱节点筛选是否需要除 `safe_dist` 外增加硬性最小间距。
5. 支撑边线输出在当前阶段使用开放边线，还是直接生成闭合外包轮廓。
