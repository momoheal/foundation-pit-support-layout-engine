# 节点与构件模型模块任务

模块目标：建立 `nodes`、`members`、`outlines`、`stats` 等新结构，同时兼容旧 layout 字段，支撑后续校核、统计、立柱和 DXF 输出。

对应需求：F3、F8、F11、F12、N3、N5。

相关文件：

- 修改：`strut_engine.py`
- 关联：`strut_validation.py`
- 关联：`main_strut.py`
- 验证：`test_strut_minimal.py`

## 最小任务清单

- [ ] 任务 1：定义节点字典结构，字段包含 `id`、`pos`、`kind`、`source`、`system`。
- [ ] 任务 2：定义构件字典结构，字段包含 `id`、`kind`、`system`、`start`、`end`、`geometry`、`width`、`material`。
- [ ] 任务 3：定义输出轮廓结构 `outlines`，字段包含 `id`、`member_id`、`layer`、`geometry`、`closed`。
- [ ] 任务 4：新增节点 ID 生成器，保证同一次求解中节点 ID 稳定且唯一。
- [ ] 任务 5：新增构件 ID 生成器，保证同一次求解中构件 ID 稳定且唯一。
- [ ] 任务 6：实现节点去重函数，按坐标容差合并重复节点。
- [ ] 任务 7：实现构件创建函数，创建构件时自动引用起点和终点节点。
- [ ] 任务 8：将腰梁点写入 `nodes`，腰梁线写入 `members`。
- [ ] 任务 9：将主撑端点和交点写入 `nodes`。
- [ ] 任务 10：将主撑线写入 `members`，类型为 `main_strut`。
- [ ] 任务 11：将角撑/八字撑端点写入 `nodes`，构件写入 `members`。
- [ ] 任务 12：将连杆/腹杆端点写入 `nodes`，构件写入 `members`。
- [ ] 任务 13：将桁架节点写入 `nodes`，弦杆和腹杆写入 `members`。
- [ ] 任务 14：将环撑与径向撑连接点写入 `nodes`。
- [ ] 任务 15：将环撑、径向撑写入 `members`。
- [ ] 任务 16：在输出中并行保留旧字段：`waling`、`corners`、`struts`、`ties`、`pillars`。
- [ ] 任务 17：新增 `issues` 字段，用于保存几何校核问题。
- [ ] 任务 18：新增空 `stats` 字段，由统计模块后续填充。
- [ ] 任务 19：写一个最小测试，确认旧字段存在且新字段存在。
- [ ] 任务 20：写一个最小测试，确认主撑交点可被识别为 `strut_cross` 节点。

## 验收清单

- [ ] 输出 layout 同时包含旧字段和新字段。
- [ ] 节点和构件 ID 稳定唯一。
- [ ] 主撑、角撑、连杆、桁架、环撑和径向撑均可表达为 `members`。
- [ ] 立柱生成可以从节点表筛选，而不是直接收集裸线段交点。
- [ ] 数据模型为后续力学分析保留构件类型和连接关系。
